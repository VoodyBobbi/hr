# HR-ассистент ООО ПОЛАТИ

Вопросно-ответный AI-ассистент (HR-менеджер по подбору персонала) для строительной компании
ПОЛАТИ. Отвечает кандидатам на вопросы по вакансиям и условиям работы строго на основе базы
знаний компании, помнит контекст диалога и помогает заполнить анкету кандидата.

Пайплайн: **данные (FAQ + база знаний) → RAG через FAISS → backend на FastAPI + GigaChat →
чат-виджет на сайте и Telegram-бот**.

За счёт автоматизации ответов на типовые вопросы кандидатов ассистент снимает до 70–80% нагрузки
с рекрутёров и сокращает среднее время первого ответа кандидату, работая 24/7 без участия
человека.

## Возможности

- **Ответы только по базе знаний компании** — модель ограничена данными из `data/`, не выдумывает
  факты (RAG-подход).
- **Память диалога** — история переписки хранится по каждому пользователю (`data/conversations/`),
  общение остаётся осмысленным.
- **Два канала** — чат-виджет на сайте и Telegram-бот, общая логика ответа.
- **Заполнение анкеты кандидата** — ассистент по одному полю собирает данные и сохраняет карточку
  в `candidats/candidates.csv` через служебные маркеры в ответе модели.
- **Логирование** — все обращения пишутся в `logs/logs.csv` (источник, вопрос, ответ, время
  ответа, статус).
- **Горячая перезагрузка** — при изменении промпта, базы знаний или индекса они подхватываются на
  лету без перезапуска сервера.

## Структура проекта

```
HR/
├── backend/
│   ├── app.py            # FastAPI-приложение: отдаёт сайт, маршруты /chat и /health
│   ├── assistant.py      # ядро: RAG-поиск, сборка промпта, вызов GigaChat, память, маркеры
│   ├── build_index.py    # сборка FAISS-индекса из data/faqs.json (эмбеддинги)
│   ├── rag_index.py      # загрузка индекса и поиск ближайших векторов (FAISS)
│   ├── candidates.py     # работа с карточками кандидатов (candidats/candidates.csv)
│   ├── telegram_bot.py   # Telegram-бот на том же ядре ассистента
│   └── logger.py         # запись обращений в logs/logs.csv
├── data/
│   ├── faqs.json             # пары «вопрос — ответ» (источник для векторного индекса)
│   ├── knowledge_base.json   # структурированная база знаний компании (в системном промпте)
│   ├── system_prompt.md      # системный промпт (роль и правила ассистента)
│   ├── faiss_index.bin       # готовый FAISS-индекс (создаётся build_index.py)
│   ├── faqs_metadata.npy      # метаданные векторов (вопрос/ответ)
│   └── faqs_hash.txt         # хэш faqs.json для пропуска лишней пересборки индекса
├── frontend/
│   ├── index.html        # страница сайта с чат-виджетом в правом нижнем углу
│   └── background.png     # фон страницы
├── candidats/            # карточки кандидатов и привязка сессий (создаётся автоматически)
├── logs/                 # логи обращений (создаётся автоматически)
├── run_all.py            # запуск сайта и Telegram-бота одной командой
└── requirements.txt      # зависимости Python
```

## Как это работает

1. **Данные.** Частые вопросы лежат в `data/faqs.json` в формате `{"question": ..., "answer": ...}`.
   Развёрнутая структурированная информация о компании, вакансиях и условиях — в
   `data/knowledge_base.json`. Поведение ассистента задаётся в `data/system_prompt.md`.
2. **Индекс.** `build_index.py` превращает каждую пару «вопрос — ответ» в вектор моделью
   `paraphrase-multilingual-MiniLM-L12-v2` (sentence-transformers) и сохраняет FAISS-индекс
   (`faiss_index.bin`) вместе с метаданными (`faqs_metadata.npy`).
3. **Запрос.** При сообщении пользователя `assistant.py` векторизует вопрос, ищет в FAISS похожие
   FAQ, добавляет их и полную базу знаний в контекст, подмешивает историю диалога и системный
   промпт, после чего вызывает **GigaChat**.
4. **Анкета.** Модель может дописывать в конец ответа служебные маркеры
   (`###SAVE_FIELD:поле=значение###`, `###LAW_ACK###`, `###CARD_CONFIRMED###`) — они вырезаются из
   ответа пользователю и сохраняют данные в карточку кандидата.
5. **Интерфейс.** `app.py` отдаёт сайт (`frontend/index.html`) и принимает запросы на `POST /chat`;
   `telegram_bot.py` использует то же ядро для Telegram.

## Требования

- Python 3.10+ (рекомендуется 3.11)
- Доступ к GigaChat API (переменная `GIGACHAT_CREDENTIALS`)
- Токен Telegram-бота — только если нужен Telegram-канал (`TELEGRAM_BOT_TOKEN`)

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
GIGACHAT_CREDENTIALS=ваш_ключ_авторизации_GigaChat
TELEGRAM_BOT_TOKEN=токен_вашего_бота   # опционально, только для Telegram
```

Файл `.env` не коммитится (см. `.gitignore`).

## Установка и запуск

```bash
# 1. Клонировать репозиторий и перейти в него
git clone https://github.com/VoodyBobbi/HR.git
cd HR

# 2. Создать и активировать виртуальное окружение
python3 -m venv venv
source venv/bin/activate        # Linux/macOS
# .\venv\Scripts\activate       # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать .env с ключами (см. раздел выше)

# 5. Собрать векторный индекс из data/faqs.json
python -m backend.build_index
# принудительная пересборка: python -m backend.build_index --force

# 6. Запустить сайт (FastAPI + чат-виджет)
uvicorn backend.app:app --host 0.0.0.0 --port 8000
```

Откройте <http://localhost:8000> — в правом нижнем углу появится иконка чата.

### Запуск сайта и Telegram-бота вместе

```bash
python run_all.py
```

Поднимет сайт на `http://localhost:8000` и Telegram-бота (нужен `TELEGRAM_BOT_TOKEN`).
Остановить всё — `Ctrl+C`.

## API

| Метод | Путь      | Описание                                                        |
|-------|-----------|----------------------------------------------------------------|
| GET   | `/`       | Главная страница сайта с чат-виджетом                           |
| POST  | `/chat`   | Отправить сообщение ассистенту                                   |
| GET   | `/health` | Проверка работоспособности сервиса (`{"status": "ok"}`)         |

Пример запроса к `/chat`:

```json
{
  "message": "Какие у вас есть вакансии?",
  "top_k": 3,
  "session_id": "необязательный-идентификатор-диалога"
}
```

Ответ:

```json
{
  "answer": "...",
  "context": [ ... ],
  "session_id": "..."
}
```

Если `session_id` не передан, он генерируется автоматически и возвращается в ответе — используйте
его в следующих запросах, чтобы сохранялась память диалога.

## Обновление базы знаний

- Изменили `data/faqs.json` — пересоберите индекс: `python -m backend.build_index`.
- Изменили `data/knowledge_base.json` или `data/system_prompt.md` — пересборка не нужна, они
  подхватываются на лету при следующем запросе.

## Деплой

1. На сервере с Python 3.10+ клонируйте репозиторий.
2. Создайте виртуальное окружение **на самом сервере** (окружение из Windows на Linux не
   запустится) и установите зависимости.
3. Создайте `.env` с ключами.
4. Соберите индекс: `python -m backend.build_index`.
5. Запустите под процесс-менеджером (`screen`/`systemd`/`tmux`):
   `uvicorn backend.app:app --host 0.0.0.0 --port 8000`.
